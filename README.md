# IT Operations SQL Project

## Overview
This is a relational database simulating an enterprise IT operations environment, designed to demonstrate my skills in SQL querying, data modeling, and business/data analytics skills.

## Database Schema
The database consists of 9 tables: departments, employees, vendors, asset_categories, assets, software_licenses, license_assignments, tickets, purchase_orders

## Business Questions Answered By Queries
- Vendor spend vs asset health: Which vendors are we spending the most with, and how many of their assets are active vs problematic?
- Software license utilization/identify waste: Which licenses are underutilized (below 80% usage)?
- Total asset value by category: How much have we spent on laptops vs servers vs networking?
- Assets coming off warranty within the next year: What do we need to budget to replace or extend coverage on?
- Which employees have more than one asset assigned?
- IT technician workload and resolution time: Who has the highest workload and fastest resolution time?
- Tickets submitted by department: Which department has the most IT problems?
- IT asset count and value by department: Which department has the most assets assigned to its employees?

## Conclusion
This project simulates a real-world IT operations environment for a company with employees across five departments and multiple office locations. The goal was to design a relational database from scratch, populate it with realistic data, and use SQL to answer meaningful business questions that an IT analyst or data analyst would face on a day-to-day basis. Key business insights uncovered through this project include identifying underutilized software licenses representing potential cost savings, flagging assets approaching warranty expiration for budget planning, and measuring IT 
technician performance through ticket resolution metrics.

This project was built independently as part of my portfolio to demonstrate hands-on SQL and data modeling skills applicable to roles in data analytics, MIS, and IT operations.
